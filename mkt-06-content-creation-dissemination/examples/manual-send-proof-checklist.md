# MKT-06 Manual Send Proof Checklist

Use this checklist whenever MKT-06 distribution is executed manually through
Reach, Gmail, Outlook, Beefree, or another non-automated send path.

## Before Sending

- [ ] Run packet exists under `examples/runs/`.
- [ ] Audience segment or CSV is named and saved.
- [ ] Each lead/company has been checked against a current public source.
- [ ] Email copy uses approved positioning and contains no unresolved merge placeholders.
- [ ] CTA destination is confirmed or intentionally omitted.
- [ ] Footer, sender identity, and unsubscribe/privacy handling are compliant for the channel used.
- [ ] Send date and timing are checked against weekends and U.S. holidays.

## Send Proof

Capture at least one of the following:

- campaign/send ID
- platform screenshot
- export showing sent status
- timestamped send log
- manual send sheet with recipient count and operator initials

## Performance Proof

Within 24-72 hours, capture:

- open count/rate
- click count/rate, if available
- reply count
- bounce count
- unsubscribe/spam count, if available
- booked call or next-step count
- notes on any manual friction

## Minimum Pass Standard

A manual send can be counted as MKT-06 distribution proof only when it has:

- recipient count
- send timestamp or send window
- channel/tool named
- proof location
- first performance signal or a scheduled follow-up time to capture it
