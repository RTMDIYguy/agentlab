# Agent Consolidation Blueprint

## Purpose

This document is the source instruction set for any agent working on the Uncle Robert Consulting operating system. The goal is not to redesign the business from scratch. The goal is to consolidate, activate, and maintain the system that already exists inside `AI Native Agency Deepened`.

The agent must reduce confusion, avoid duplicating files, preserve recovered material, and create a clean operating layer for `URC`, `Tactix`, `Bootstrapper Capital`, and `Bootstrapper's Guide to the World`.

## Primary Objective

Create and maintain a working executive layer on top of the existing departmental system.

The agent should:
- identify source-of-truth files
- avoid creating duplicate strategy documents
- sort recovered business files into the proper business structure
- preserve archives separately from active files
- connect the book and Bootstrapper Capital into the current sales and marketing machine
- establish a weekly operating cadence using the files already present

## Business Structure

Use this hierarchy unless explicitly instructed otherwise:

- `URC` = primary business and operating system owner
- `Tactix` = fulfillment and Upwork delivery arm
- `Bootstrapper Capital` = audience, founder community, event, and low-ticket conversion funnel
- `Bootstrapper's Guide to the World` = authority asset and lead-generation asset

## Non-Negotiable Rules

- Do not redesign the business from zero.
- Do not replace working spreadsheets with new versions unless there is a clear defect.
- Do not move recovered files directly into active folders without triage.
- Do not use `.zip`, `backup`, `prompt result`, `reformat`, or `update needed` files as source-of-truth files unless there is no cleaner live version.
- Treat `D:\Recover` as raw recovery archive material.
- Prefer consolidation over creation.
- Preserve business history, but separate `active`, `archive`, and `review` materials clearly.

## Required Folder States

The agent should maintain the following states conceptually, whether or not they exist yet as folders:

- `Active`
- `Archive`
- `Review`
- `Recovered Intake`

If the agent creates folders, use those names plainly and keep them inside the relevant business area.

## Recovered Drive Intake

Recovered material currently exists in:

- `D:\Business Docs Recovered`
- `D:\Misc Recovered`
- `D:\Recover`

### Recovery Intake Rules

- `D:\Recover` is the raw archive. Do not use it as the live working set.
- Prioritize `D:\Business Docs Recovered` before `D:\Misc Recovered`.
- Never delete recovered files during initial triage.
- Move only reviewed business documents into `Working Docs`.
- If duplicates are found, keep the most complete, most recent, and best-named version active; move the others to archive or review.

### Intake Categories

Each recovered file should be sorted into one of these buckets:

- `URC`
- `Tactix`
- `Bootstrapper Capital`
- `Book`
- `Sales`
- `Marketing`
- `Finance`
- `Operations`
- `Fulfillment`
- `AfterCare`
- `Culture`
- `Client Delivery`
- `Personal`
- `Archive`
- `Review`

### Intake Workflow

For every recovered business file:

1. Determine whether it is business or personal.
2. If personal, leave it outside the active business structure.
3. If business, determine the best destination category.
4. Check whether a matching file already exists in `AI Native Agency Deepened`.
5. If a better live version already exists, move the recovered file to `Archive` or `Review`.
6. If the recovered file is clearly newer, more complete, or missing from the active layer, place it in `Review` first and flag it for merge.
7. Do not silently overwrite active files.

## Executive Command Layer

The agent should maintain one lightweight executive layer above the departmental files.

This layer should answer:

- What are we selling right now?
- Where are leads coming from?
- What content is publishing this week?
- What invoices are due or overdue?
- What client work is at risk?
- What events or community activities are upcoming?
- How is the book feeding the funnel?

If a new file is needed for this purpose, create only one concise command document or workbook, not several.

Suggested name:

- `Executive Command Center`

## Cross-Brand Funnel

The agent should maintain the following conversion map:

- content and outreach create awareness
- the book builds trust and authority
- Bootstrapper Capital creates community entry and live interaction
- URC provides paid advisory and implementation
- Tactix supports fulfillment and delivery capacity

### Offer Ladder

The offer ladder should be reflected in sales and marketing materials:

- `Bootstrapper's Guide to the World`
- free founder conversation, roundtable, or event
- workshop or bootcamp
- monthly continuity offer
- URC consulting or implementation sprint
- Tactix-supported execution where appropriate

## Weekly Operating Cadence

The agent should structure work around this weekly rhythm:

- Monday: sales pipeline, opportunities, follow-up, proposals
- Tuesday: marketing, content, book promotion, outreach
- Wednesday: Bootstrapper Capital community, event planning, partnerships
- Thursday: fulfillment, aftercare, delivery quality, client health
- Friday: finance review, KPI review, planning, cleanup

## Department Expectations

### Sales

The agent should ensure that:

- at least one sales pipeline tracker is current
- funding and investor opportunities are separated from normal client sales when useful
- onboarding trackers are actually used after deal close
- the activity dashboard is updated regularly

### Marketing

The agent should ensure that:

- content production is tied to offers
- outreach and engagement are tracked
- reviews and referrals are intentionally requested
- book-related content is part of the content tracker

### Finance

The agent should ensure that:

- AR and AP are visible
- pricing and expenses are reviewed
- the finance KPI dashboard reflects current business reality
- Frappe remains the transactional system of record where applicable

### Operations

The agent should ensure that:

- the SOP registry stays current
- the strategy and OKR tracker reflects current priorities
- the automation registry reflects actual tools in use
- quality issues and experiments are captured rather than scattered

### Fulfillment and AfterCare

The agent should ensure that:

- client health is visible
- post-sale support is not forgotten
- fulfilled engagements generate referrals, testimonials, or upsell opportunities

## Book Integration Rules

The book must not remain isolated.

The agent should connect the book into:

- content creation
- event invitations
- founder roundtables
- workshop offers
- discovery calls

The book should be treated as an authority asset first and a direct-sale product second.

## What Success Looks Like

The system is considered consolidated when:

- each department has a clear source-of-truth file set
- recovered business files are sorted out of raw recovery folders
- duplicates are archived rather than left mixed with live files
- there is one executive command layer
- the weekly cadence is being followed
- the book clearly feeds the sales and marketing funnel

## First Actions For Any Agent

When an agent starts on this system, it should begin in this order:

1. Review the strategy anchors.
2. Confirm the active file set for each department.
3. Inventory recovered business files in `D:\Business Docs Recovered`.
4. Create a triage list for `keep active`, `archive`, and `review`.
5. Build or update the executive command layer.
6. Align current tasks to the weekly cadence.
7. Report back with:
   - active files
   - archive candidates
   - recovered files needing manual judgment
   - the next three implementation actions
