# URC Agent Execution Checklist

## Purpose

This checklist turns the URC operating architecture and 90-day plan into a direct execution sequence for an agent.

Read these first:

1. `docs/operations/urc-v1-operating-architecture.md`
2. `docs/operations/urc-90-day-implementation-plan.md`

Do not redesign the business from scratch. Execute the plan in a lightweight, practical way.

## Execution Rules

- prefer implementation over brainstorming
- use existing systems before adding new tools
- keep `Microsoft 365` as the preferred operating backbone when paid access is
  available; when paid access is unavailable, use free Microsoft accounts,
  local files, repo-tracked CSV/Markdown, Reach, Gmail, or other free/low-cost
  bridges and record the fallback
- keep `Frappe` as the finance system of record
- keep `Notion` limited to lightweight dashboard and planning use
- treat the book as a funnel asset, not a standalone product
- keep `URC` as the main brand
- keep `Bootstrapper Capital` as the audience and event funnel
- keep `Tactix` as the fulfillment arm

### 1. Confirm Business Hierarchy

- verify `URC` is positioned as the primary business
- verify `Bootstrapper Capital` is positioned as the funnel/community layer
- verify `Tactix` is positioned as the execution arm
- verify the book is positioned as an authority asset with a clear next step

### 2. Confirm Tool Split

- confirm whether paid `Microsoft 365` is available; if not, use the approved
  free-tool bridge for email, calendar, files, forms, and collaboration
- confirm `Frappe` usage for invoices and payment status
- confirm `Notion` is not being used as the main archive
- note any tool overlap or unnecessary duplication

### 3. Define The Active Offer Ladder

- book
- founder session or roundtable
- workshop or bootcamp
- continuity offer
- consulting sprint or advisory

Output:

- one concise offer ladder summary

### 4. Build Or Update CRM-Lite

- confirm where the lead tracker lives
- ensure it tracks source, stage, next step, owner, and follow-up date
- ensure it can distinguish book, event, referral, and Upwork leads

Output:

- one CRM-lite recommendation or update plan

### 5. Build Or Update The Book Funnel

- define one CTA for the book
- define one next step after book interest
- define how book content becomes LinkedIn and Instagram content
- define how book traffic feeds an event, form, or workshop

Output:

- one book-to-client funnel summary

### 6. Establish Weekly Rhythm

- Monday: sales and pipeline
- Tuesday: marketing and book promotion
- Wednesday: Bootstrapper Capital and events
- Thursday: fulfillment and aftercare
- Friday: finance and KPI review

Output:

- one weekly operating checklist

### 7. Define Phase 1 Actions

Focus on days 1-30:

- confirm the operating backbone
- create one executive dashboard
- define one offer ladder
- define one book CTA
- create one file and folder structure if still missing

Output:

- top 5 actions for days 1-30

### 8. Define Phase 2 Actions

Focus on days 31-60:

- launch one founder-facing funnel
- start one recurring event rhythm
- standardize CRM-lite updates
- standardize invoicing in `Frappe`
- turn the book into reusable content

Output:

- top 5 actions for days 31-60

### 9. Define Phase 3 Actions

Focus on days 61-90:

- launch the continuity offer
- launch one flagship workshop or bootcamp
- build the referral loop
- review monthly results
- identify what to scale

Output:

- top 5 actions for days 61-90

### 10. Report Only What Matters

At the end of a run, report:

- what already exists
- what was clarified
- what is still missing
- the next 3 implementation actions

Do not produce long speculative strategy memos unless asked.

## Success Condition

This checklist is successful when the business has:

- a clear brand hierarchy
- a practical offer ladder
- a defined book funnel
- a lightweight CRM process
- a weekly operating rhythm
- a visible 90-day execution path
