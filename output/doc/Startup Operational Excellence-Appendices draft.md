# Appendix A: Operational Audit Question Bank

Use this appendix when the business feels heavier than it should and you need
to see what is actually happening.

Do not try to answer every question in one sitting. Choose one customer path,
one workflow, or one recurring problem. Then use the questions that help you
see the truth more clearly.

## Follow The Customer Request

1. Where does the customer request first arrive?
2. Who sees it first?
3. Where is the customer's name, contact information, and need captured?
4. What happens if the founder is busy when the request arrives?
5. What information must be collected before the next step can happen?
6. Where does that information live?
7. What step depends on memory?
8. What step depends on a message thread, note, or old email?
9. What does the customer see while waiting?
10. What confirms that the request has moved forward?

## Where Does The Work Wait?

1. Where does the work pause most often?
2. Is the pause caused by missing information, unclear ownership, tool friction,
   customer delay, founder review, or cash timing?
3. How long does the pause usually last?
4. Who notices the pause first?
5. Does the customer know what is happening during the pause?
6. Does the team know what is needed to restart the work?
7. Is the wait point visible in a shared system?
8. What one field, checklist, owner, or reminder would make the wait easier to
   manage?

## What Lives Only In Someone's Head?

1. Which customer details depend on memory?
2. Which exceptions are known by one person only?
3. Which passwords, files, links, templates, or instructions are hard to find?
4. Which decision rules have never been written down?
5. Which quality standard is understood but not documented?
6. Which promise would be difficult for someone else to fulfill if the founder
   were unavailable?
7. What should be moved from memory into a visible system this week?

## Tool Check: Helpful System Or Expensive Habit?

1. What job is this tool supposed to do?
2. Does the team trust the information inside it?
3. Does it reduce work, or does it create another place to check?
4. Does it connect to the customer path, delivery path, or money path?
5. Is it solving a proven problem or preserving an old hope?
6. Would the business break if this tool disappeared for thirty days?
7. Is this tool still runway, or has it become clutter?

# Appendix C: Resource Allocation Scorecard

Use this scorecard when the business has more ideas than capacity.

Score each active project, offer, or commitment from 1 to 5. A 1 means weak or
unclear. A 5 means strong and obvious.

| Item | Revenue Potential | Customer Value | Strategic Fit | Delivery Capacity | Cash Impact | Learning Value | Total |

|  |  |  |  |  |  |  |  |
|  |  |  |  |  |  |  |  |
|  |  |  |  |  |  |  |  |

## Interpretation

High score, high capacity:

Do this deliberately. Assign the next action and owner.

High score, low capacity:

Protect the opportunity, but reduce scope or delay until capacity is real.

Low score, high effort:

Stop, pause, or redesign. Effort is not proof of value.

Unclear score:

Run a small test before making a large commitment.

## Capacity Questions

1. Who has to act for this to move forward?
2. What existing promise would be delayed if we say yes?
3. What decision does the founder need to make?
4. What could be delegated if the standard were clearer?
5. What can be scheduled, reduced, paused, or removed?
